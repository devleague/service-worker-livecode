Service Worker Background Sync Example

Background sync allows HTTP requests to be queued in the background in situations where network in not available.
The service worker has been registered which means all the static files have been cached beforehand.
The form below will take the user input and set that value into indexedDB. Then it will send an event to the service worker to trigger a background POST request to our server when there's a network available. Things to notice: The indexedDB section of the "Application" tab in Chrome Dev Tools should have an entry for "bg-input-key", if there's no network, a request was not sent out in the "Network" tab, and finally once a network connection is restored, a POST request will be made to our server with the user input as the body.
